- ❌ **일반 와이파이 연결 시**: SSH 접속 불가 (현재 IP 203.229.216.9가 ACG에 등록되지 않음)

## 원인 분석

### 1. 네이버 클라우드 ACG (Access Control Group) 설정

네이버 클라우드의 ACG는 **방화벽 규칙**으로, 특정 IP에서만 접근을 허용합니다.

#### 현재 설정 (추정)
```
포트 1025:
- ACG 규칙: 0.0.0.0/0 (전체 허용) 또는 특정 IP만 허용
- 실제로는 핫스팟 IP만 허용되어 있음

포트 22:
- ACG 규칙: 특정 IP만 허용
- 현재 IP가 허용 목록에 없음
```

### 2. IP 주소 차이

#### 핫스팟 IP
- 핫스팟을 사용할 때는 **모바일 통신사 IP**를 사용
- 이 IP가 ACG에 등록되어 있음

#### 일반 와이파이 IP
- 일반 와이파이를 사용할 때는 **다른 IP**를 사용
- 이 IP가 ACG에 등록되어 있지 않음

### 3. 포트 포워딩 설정

네이버 클라우드에서 포트 포워딩을 설정했을 가능성:
- 외부 포트 1025 → 내부 포트 22로 포워딩
- 하지만 ACG가 특정 IP만 허용하도록 설정됨

## 해결 방법

### 방법 1: ACG에 현재 IP 추가 (권장)

1. **현재 IP 확인**
   ```bash
   curl ifconfig.me
   # 또는
   curl ipinfo.io/ip
   ```

2. **네이버 클라우드 콘솔 접속**
   - 네이버 클라우드 플랫폼 콘솔 로그인
   - 서버 → ACG 메뉴 이동

3. **ACG 규칙 수정**
   - 포트 1025 인바운드 규칙 확인
   - 현재 IP 주소 추가
   - 또는 `0.0.0.0/0`으로 변경 (보안 주의)

### 방법 2: 포트 22 직접 사용

1. **ACG에서 포트 22 규칙 확인**
   - 현재 IP를 포트 22 허용 목록에 추가

2. **스크립트 수정**
   ```bash
   # deploy_to_different_port.sh 수정
   SERVER_PORT="22"  # 1025 → 22로 변경
   ```

### 방법 3: 동적 IP 대응 (고급)

#### Option A: ACG에 여러 IP 등록
- 집, 회사, 핫스팟 등 사용하는 모든 IP를 ACG에 등록

#### Option B: VPN 사용
- 고정 IP를 가진 VPN 서버를 통해 접속

#### Option C: 네이버 클라우드 Bastion Host 사용
- 네이버 클라우드의 Bastion Host 기능 사용

## 현재 설정 확인 방법

### 1. 현재 IP 확인
```bash
# 현재 사용 중인 공인 IP 확인
curl ifconfig.me
```

### 2. SSH 접속 테스트
```bash
# 포트 1025 테스트
ssh -i ~/.ssh/nimso2026.pem -p 1025 root@27.96.129.114 "echo '연결 성공'"

# 포트 22 테스트
ssh -i ~/.ssh/nimso2026.pem -p 22 root@27.96.129.114 "echo '연결 성공'"
```

### 3. 네이버 클라우드 콘솔에서 확인
- **서버** → **서버 관리** → 해당 서버 선택
- **ACG** 탭에서 인바운드 규칙 확인
- 포트 1025, 22의 허용 IP 목록 확인

## 참고 사항

### 보안 고려사항
- `0.0.0.0/0` (전체 허용)은 보안상 위험할 수 있음
- 가능하면 특정 IP만 허용하는 것을 권장
- SSH 키 인증을 사용 중이므로 상대적으로 안전

### 네이버 클라우드 ACG 설정 위치
1. 네이버 클라우드 플랫폼 콘솔
2. **Server** → **Server** 메뉴
3. 해당 서버 선택
4. **ACG** 탭 클릭
5. **Inbound Rule** 수정

## 요약

**왜 핫스팟에서만 접속 가능한가?**
- 핫스팟 IP가 ACG에 등록되어 있음
- 일반 와이파이 IP는 ACG에 등록되어 있지 않음
- 포트 1025는 포트 포워딩이 설정되어 있지만, ACG가 특정 IP만 허용

**해결 방법:**
- 네이버 클라우드 콘솔에서 ACG 규칙 확인
- 현재 사용 중인 IP를 ACG에 추가
- 또는 포트 1025를 `0.0.0.0/0`으로 변경 (보안 주의)
