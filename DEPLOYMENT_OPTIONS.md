# 배포 옵션 가이드

## 배포 방법

### 옵션 1: 같은 주소에 업데이트 (기본)
```bash
./deploy_current_branch.sh
```
- 주소: http://115.85.181.103:8000
- 기존 서버 업데이트

### 옵션 2: 다른 포트로 배포
```bash
./deploy_to_different_port.sh 8001
```
- 주소: http://115.85.181.103:8001
- 같은 서버, 다른 포트
- 기존 8000 포트와 별도로 실행

### 옵션 3: 다른 서버로 배포
- 새 서버 생성 후 배포
- 다른 IP 주소 사용

## 추천 방법

### develop 브랜치를 다른 포트로 배포하려면:
```bash
./deploy_to_different_port.sh 8001
```

### 결과:
- 기존: http://115.85.181.103:8000 (dev_hwan)
- 새로: http://115.85.181.103:8001 (develop)

## 주의사항

1. **ACG 설정 필요**
   - 네이버 클라우드 콘솔에서 새 포트 허용 필요
   - 예: 포트 8001 인바운드 규칙 추가

2. **방화벽 설정**
   - 스크립트가 자동으로 설정하지만 확인 필요

3. **DB 공유**
   - 같은 PostgreSQL DB 사용 (데이터 공유)
