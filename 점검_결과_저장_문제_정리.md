# 점검 결과 저장 문제 총 정리

## 📋 문제 현황

### 1. 핵심 문제: 데이터베이스 저장 실패 (500 Internal Server Error)

**에러 메시지:**
```
(psycopg2.errors.NotNullViolation) null value in column "id" violates not-null constraint
DETAIL: Failing row contains (null, test, test, 2026-01-27T16:10:00Z, test, success, {}, 2026-01-27 16:28:18.324994).
```

**발생 시점:**
- 오늘 3시 점검 시도 시 모두 실패
- 방금 실행한 점검(16:09)도 모두 실패
- OS, MariaDB, Tomcat 점검은 성공했지만 API 전송 시 500 에러

**영향:**
- 모든 점검 결과가 데이터베이스에 저장되지 않음
- 리포트 페이지에 최신 데이터가 표시되지 않음
- 가장 최근 저장된 데이터: 2026-01-22 10:24:51

---

## 🔍 원인 분석

### 문제의 근본 원인: PostgreSQL ID 컬럼 시퀀스 문제

**현재 데이터베이스 스키마:**
```sql
id | integer | not null  -- ❌ SERIAL 또는 IDENTITY가 아님
```

**SQLAlchemy 모델:**
```python
id = Column(Integer, primary_key=True, index=True, autoincrement=True)
```

**문제점:**
1. PostgreSQL에서 `autoincrement=True`만으로는 자동 증가가 작동하지 않음
2. PostgreSQL은 `SERIAL` 타입 또는 `GENERATED ALWAYS AS IDENTITY`가 필요
3. 현재 테이블은 `integer not null`로만 정의되어 있어 시퀀스가 없음
4. SQLAlchemy가 INSERT 시 id 값을 제공하지 않으면 NULL이 되어 NOT NULL 제약조건 위반

## 🛠️ 해결 방안

### 방법 1: 데이터베이스 테이블 수정 (권장) ✅

**현재 상태 확인:**
- 최대 ID: 139
- 시퀀스 존재 여부: 없음 ❌

**수정 SQL (실행 필요):**

```sql
-- 1. 시퀀스 생성
CREATE SEQUENCE IF NOT EXISTS check_results_id_seq;

-- 2. 시퀀스를 id 컬럼에 연결 (기본값 설정)
ALTER TABLE check_results 
ALTER COLUMN id SET DEFAULT nextval('check_results_id_seq');

-- 3. 시퀀스의 현재 값을 최대 id 값(139) 다음으로 설정
SELECT setval('check_results_id_seq', (SELECT MAX(id) FROM check_results));

-- 4. 다음 ID가 140부터 시작하도록 설정
SELECT setval('check_results_id_seq', 140);
```


### 방법 2: 테이블 재생성 (데이터 백업 필요)

```python
# models.py에서 SERIAL 타입 사용하도록 변경
from sqlalchemy import Sequence

id = Column(Integer, Sequence('check_results_id_seq'), primary_key=True)
```

그 후 테이블 재생성 (기존 데이터 백업 필수)

### 방법 3: SQLAlchemy 설정 변경

PostgreSQL 전용으로 명시적 시퀀스 사용:

```python
from sqlalchemy import Sequence

id = Column(
    Integer, 
    Sequence('check_results_id_seq', start=140),  # 다음 ID부터 시작
    primary_key=True
)
```
