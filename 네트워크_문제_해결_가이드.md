# 네트워크 문제 해결 가이드

## 🔍 현재 문제

- `115.85.181.103:22` 접근 불가 (포트 22 타임아웃)
- `27.96.129.114:1025` 접근 불가 (포트 1025 타임아웃)
- `27.96.129.114`는 ping은 성공 (서버는 살아있음)


### 1단계: 네이버 클라우드 보안 그룹(ACG) 설정 확인

네이버 클라우드 콘솔에서:

1. **Server → Server → nimbus-server 선택**
2. **ACG (Access Control Group) 확인**
   - 현재: `ncloud-default-acg(397934)`
3. **"규칙 보기" 버튼 클릭**
4. **인바운드 규칙 확인:**
   - SSH 포트 22가 허용되어 있는지 확인
   - 포트 포워딩 사용 시 포트 1025도 확인

### 2단계: 보안 그룹 규칙 추가 (필요 시)

**SSH 포트 22 허용:**
- 프로토콜: TCP
- 포트: 22
- 출발지: `0.0.0.0/0` (모든 IP) 또는 본인 PC 공인 IP

**포트 포워딩 사용 시 포트 1025 허용:**
- 프로토콜: TCP
- 포트: 1025
- 출발지: `0.0.0.0/0` 또는 본인 PC 공인 IP

### 3단계: 본인 PC 공인 IP 확인

```bash
curl ifconfig.me
```

이 IP를 보안 그룹 출발지에 추가

### 4단계: 서버 상태 확인

네이버 클라우드 콘솔에서:
- 서버 상태가 "운영중"인지 확인
- 서버가 정지되어 있다면 시작

---

## 🧪 해결 후 테스트

보안 그룹 설정 후:

```bash
# 방법 1: 공인 IP로 접속
ssh -i ~/.ssh/nimso2026.pem root@115.85.181.103

# 방법 2: 포트 포워딩 사용
ssh -i ~/.ssh/nimso2026.pem -p 1025 root@27.96.129.114
```
