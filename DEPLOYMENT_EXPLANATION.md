# 배포 방식 설명

## 배포 동작

### 같은 서버, 같은 주소에 업데이트

배포 스크립트를 실행하면:

1. **같은 서버에 배포**
   - 서버: `nimbus-server` (115.85.181.103)
   - 경로: `/opt/ansible-monitoring/api_server/`

2. **같은 웹 주소 사용**
   - 주소: `http://115.85.181.103:8000`
   - 기존 파일을 **덮어쓰기** (업데이트)

3. **데이터는 유지**
   - PostgreSQL DB 데이터는 그대로 유지
   - 코드만 업데이트됨

## 예시

### 배포 전
- 주소: http://115.85.181.103:8000/api/report
- 코드: dev_hwan 브랜치 버전

### 배포 후 (develop 브랜치)
- 주소: **동일** http://115.85.181.103:8000/api/report
- 코드: develop 브랜치 버전 (업데이트됨)
- 데이터: 그대로 유지

## 요약

✅ **같은 주소에 업데이트**
- 새로운 서버 생성 안 함
- 새로운 주소 생성 안 함
- 기존 서버의 코드만 업데이트

❌ **다른 주소로 배포 안 함**
- 같은 서버, 같은 포트 사용
