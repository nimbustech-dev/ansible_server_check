# 우분투 터미널 사용 가이드

## Windows에서 우분투 터미널 여는 방법

### 방법 1: WSL 우분투 사용 (로컬) ⭐ 권장

#### PowerShell 또는 명령 프롬프트에서:

```powershell
# 우분투 터미널 열기
wsl

# 또는 직접 우분투 실행
ubuntu

# 또는 특정 디렉토리에서 열기
wsl -d Ubuntu
```

#### Windows 터미널에서:
1. Windows 터미널 실행 (Win + X → Windows 터미널)
2. 드롭다운 메뉴에서 "Ubuntu" 선택
3. 또는 새 탭에서 "Ubuntu" 선택

#### 바로가기로 실행:
- 시작 메뉴에서 "Ubuntu" 검색 후 실행

---

### 방법 2: 원격 우분투 서버에 SSH 연결

#### SSH로 원격 서버 접속:

```bash
# 기본 형식
ssh 사용자명@서버IP주소

# 예시
ssh root@192.168.0.100
ssh ubuntu@115.85.181.103

# SSH 키 사용
ssh -i ~/.ssh/키파일명 사용자명@서버IP주소

# 예시 (nimbus 서버)
ssh -i ~/.ssh/nimsou2021 root@115.85.181.103
```

#### Windows에서 SSH 사용:
1. **PowerShell에서 직접 사용** (Windows 10/11 기본 포함)
   ```powershell
   ssh root@192.168.0.100
   ```

2. **WSL 우분투에서 사용** (더 많은 옵션)
   ```bash
   wsl
   ssh root@192.168.0.100
   ```

3. **PuTTY 사용** (GUI 도구)
   - PuTTY 다운로드: https://www.putty.org/
   - Host Name: 서버 IP 주소
   - Port: 22 (기본)
   - Connection type: SSH
   - Open 클릭

## 현재 프로젝트에서 우분투 사용하기

### WSL 우분투에서 프로젝트 접근:

```bash
# 1. WSL 우분투 열기
wsl

# 2. Windows 파일 시스템 접근
cd /mnt/c/ansible_server_check

# 또는 Windows 경로를 WSL에서 사용
cd /mnt/c/Users/사용자명/ansible_server_check
```

### WSL에서 Ansible 실행:

```bash
# WSL 우분투에서
cd /mnt/c/ansible_server_check

# Ansible 설치 확인
ansible --version

# 플레이북 실행
ansible-playbook -i inventory nimbus_check/os_check.yml
```

## WSL 우분투 설정 (처음 사용 시)

### 1. WSL 우분투 초기 설정:

```bash
# WSL 우분투 열기
wsl

# 사용자명과 비밀번호 설정 (처음 실행 시)
# (이미 설정되어 있으면 생략)
```

### 2. 필요한 패키지 설치:

```bash
# 패키지 업데이트
sudo apt update
sudo apt upgrade -y

# Ansible 설치
sudo apt install -y ansible

# Python 및 pip 설치
sudo apt install -y python3 python3-pip

# Git 설치
sudo apt install -y git
```

### 3. 프로젝트 파일 접근:

```bash
# Windows 파일 시스템은 /mnt/c/ 경로로 접근
ls /mnt/c/

# 프로젝트 디렉토리로 이동
cd /mnt/c/ansible_server_check
```

## 원격 서버 접속 예시

### nimbus 서버 접속 (프로젝트 문서 참고):

```bash
# WSL 우분투에서
wsl

# SSH 키가 있는 경우
ssh -i ~/.ssh/nimsou2021 root@115.85.181.103

# 또는 비밀번호 사용 (보안상 권장하지 않음)
ssh root@115.85.181.103
```

### SSH 키 설정:

```bash
# 1. SSH 키 생성 (없는 경우)
ssh-keygen -t rsa -b 4096 -f ~/.ssh/naver_cloud_key

# 2. 공개키를 서버에 복사
ssh-copy-id -i ~/.ssh/naver_cloud_key.pub 사용자명@서버IP

# 3. 접속 테스트
ssh -i ~/.ssh/naver_cloud_key 사용자명@서버IP
```

## 유용한 팁

### 1. WSL과 Windows 파일 공유:
- Windows 파일: `/mnt/c/` 경로로 접근
- WSL 파일: Windows 탐색기에서 `\\wsl$\Ubuntu\` 경로로 접근

### 2. 환경 변수 공유:
- WSL에서 Windows 환경 변수 접근 가능
- Windows에서 WSL 명령 실행: `wsl 명령어`

### 3. 네트워크 접근:
- WSL은 Windows와 같은 네트워크 사용
- localhost는 WSL과 Windows 간 공유됨

## 문제 해결

### WSL이 실행되지 않을 때:

```powershell
# PowerShell에서 (관리자 권한)
wsl --update
wsl --set-default Ubuntu
```

### SSH 연결이 안 될 때:

```bash
# 연결 테스트
ping 서버IP주소

# SSH 디버그 모드
ssh -v 사용자명@서버IP주소

# 포트 확인
telnet 서버IP주소 22
```

### 파일 경로 문제:

```bash
# Windows 경로를 WSL 경로로 변환
# C:\ansible_server_check → /mnt/c/ansible_server_check

# 현재 경로 확인
pwd

# Windows 경로 확인
wslpath -w $(pwd)
```
