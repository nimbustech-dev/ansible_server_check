# SSH 접속 문제 진단 및 해결

## 🔍 현재 상황

- ✅ 보안 그룹(ACG): 포트 22가 `0.0.0.0/0`에서 허용됨
- ❌ SSH 접속: 타임아웃 (포트 22 접근 불가)
- ✅ 포트 포워딩 IP: `27.96.129.114` ping 성공
- ❌ 포트 포워딩 포트 1025: 접근 불가

## 🔎 가능한 원인

### 1. 서버 상태 확인 필요

네이버 클라우드 콘솔에서:
- 서버 상태가 "운영중"인지 확인
- 서버가 정지되어 있다면 시작

### 2. 서버 내부 방화벽 확인

서버 콘솔에서 확인:
```bash
# CentOS 7의 경우
systemctl status firewalld
# 또는
iptables -L -n
```

### 3. 네트워크 라우팅 문제

- 공인 IP `115.85.181.103`이 실제로 서버에 연결되어 있는지 확인
- 포트 포워딩 설정이 올바른지 확인

### 4. 서버가 다른 네트워크에 있을 수 있음

- VPC 환경인 경우 네트워크 설정 확인 필요

## ✅ 해결 방법

### 방법 1: 서버 상태 확인 (가장 먼저)

네이버 클라우드 콘솔에서:
1. Server → Server → nimbus-server 선택
2. 서버 상태 확인
3. "정지" 상태라면 "시작" 버튼 클릭

### 방법 2: 서버 콘솔에서 방화벽 확인

서버 접속 콘솔에서:
```bash
# 방화벽 상태 확인
systemctl status firewalld

# 방화벽이 실행 중이라면 SSH 포트 허용
firewall-cmd --permanent --add-service=ssh
firewall-cmd --reload

# 또는 iptables 확인
iptables -L -n | grep 22
```

### 방법 3: 포트 포워딩 재확인

네이버 클라우드 콘솔에서:
1. 서버 상세 정보 확인
2. 포트 포워딩 정보 확인:
   - 서버 접속용 공인 IP: `27.96.129.114`
   - 외부 포트: `1025`
3. 포트 포워딩이 올바르게 설정되어 있는지 확인

### 방법 4: 네트워크 인터페이스 확인

서버 콘솔에서:
```bash
# 네트워크 인터페이스 확인
ip addr show

# SSH 서비스 상태 확인
systemctl status sshd
```

## 🧪 테스트 순서

1. **서버 상태 확인** (네이버 클라우드 콘솔)
2. **서버 콘솔에서 방화벽 확인**
3. **포트 포워딩 설정 확인**
4. **SSH 접속 재시도**
